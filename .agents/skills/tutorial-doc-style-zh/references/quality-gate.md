# 质量门

运行 `SKILL.md` 中的 Multi-Agent Quality Gate 时读取本文件。质量门优先判断读者阻断：只要存在严重 reader-experience blocker，即使分数很高，教程也不能通过。

## 1. 评审材料包

启动任何 reviewer 前，提供：

- 已确认的 teaching brief，包括目标读者、边界、排除范围、深度、内容目录、图表计划和测评重点；
- brief 未说明时的默认读者：有基础认知和推导能力但首次接触该领域的大学生；
- 目标文档全文，而不只是摘录；
- 图片清单，包括路径、图注、预期教学目的和附近正文；
- 外部事实依赖的来源说明或引用列表；
- 下方通用阻塞项和对应角色提示词。

## 2. 通用评审规则

- 以真实学习者身份从头到尾阅读文档，不要只扫孤立事实；
- 判断每个概念、公式、图表和结论是否已经被前文充分铺垫；
- 尽可能给出章节或行级发现；
- 阻塞项必须判为 fail，不能被高分抵消；
- 不要因为主题困难、篇幅长或图很多就放宽通过标准。

通用阻塞项：

- 导读在读者尚未形成学习动机前，突然引入未解释术语、公式、工具或子主题；
- 正文使用“回到”“如前所述”“因此”等承接词，但前文没有真正建立必要上下文；
- 图像引入附近正文没有铺垫或解释的对象、公式、标签或论断；
- 关键结论以清单或压缩断言突然出现，缺少推理、例子或可记忆锚点；
- 章节像知识点拼贴，而不是连续教学路径；
- 学习测评考察文章边界、作者决策、排除主题或过程选择，而不是读者应理解的知识；
- 学习测评题目荒谬、玩笑化、过于琐碎、有歧义，或偏离既定学习目标。

## 3. 角色提示词

Content accuracy reviewer：

```text
请审阅完整教程的事实准确性、来源支撑、术语和概念顺序。检查论断是否由给定来源支撑，或是否清楚标注为本地解释。标出任何事实错误、缺少来源的易漂移断言、误导性简化，或会让目标读者困惑的概念顺序问题。
```

Reader experience reviewer：

```text
除非 teaching brief 指定了不同读者，否则请站在“有基础认知和推导能力但首次接触该领域的大学生”视角完整阅读教程。检查文章是否亲切、导读是否建立学习动机、公式和例子是否先铺垫再复用、转场是否逻辑成立、图文是否共同教学，以及每个主要章节是否留下稳定记忆点。任何 reader-blocking 问题都必须判为 fail。
```

Visual quality reviewer：

```text
根据图片清单、图注、教学目的和附近正文审查每张图。检查每张图是否有明确教学任务、标签和对象是否技术正确、正文是否铺垫并解释该图、布局是否可读，以及是否出现视觉反模式。若图片文件不可用，报告限制，并继续审查图表意图和正文整合。
```

Assessment author：

```text
阅读完整教程和 teaching brief 后再起草学习测评。题目只能考察知识理解、迁移判断、常见误区或机制推理。不要询问文章为什么排除某主题、作者如何设计教程，或其他写作决策类元问题。题型以单选和多选为主，干扰项必须可信。
```

Assessment reviewer：

```text
审查最终学习测评的正确性、覆盖度、歧义、难度平衡和学习价值。拒绝关于文章范围或作者决策的元问题，拒绝与读者知识无关的问题，拒绝无意义或玩笑式干扰项，拒绝不能解释所考察区分点的答案解析。
```

## 4. 输出格式

每个 reviewer 返回：

```markdown
Score: <0-100>
Pass: <yes/no>
Blocking issues:
- <section/line>: <issue and why it blocks the target reader>
Line/section findings:
- <section/line>: <specific problem>
Required revisions:
- <specific change required before pass>
Residual risks:
- <non-blocking concern, or "None">
```

主 agent 必须修正所有阻塞项，并重新运行未通过的评审角色后才能定稿。
