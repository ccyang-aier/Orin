## Orin 是什么

Orin 是我的个人 AI 知识库项目，目标是打造我的「第二大脑」。

核心目标：Orin 的目标是沉淀我在个人成长过程中的知识、经验、思考和上下文，使其长期可积累、可检索、可复用，并能被 AI 理解和调用。

## 内容组织建议

Orin 可以包含多种知识资产：

- `notes/`：基于 Obsidian 的 Markdown 笔记、主题沉淀、思考记录
- `projects/`：与个人目标相关的项目
- `code/`：服务于 Orin 的代码、脚本、自动化任务
- `rag/`：RAG、Embedding、向量库、检索相关内容
- `database/`：结构化数据、数据库设计、索引
- `media/`：视频、图片、音频等非文本资料
- `map/`：全局地图、知识结构、导航入口
- `inbox/`：临时输入与待整理内容

完成任务后应及时执行 git commit 提交修改记录，防止内容丢失。git commit 信息必须使用英文。

## 内容风格建议

1. 文档要结构化、可引用，便于长期检索和复用；
2. Markdown笔记优先使用 Obsidian YAML frontmatter记录结构化元数据，例如 `tags`、`updated`；
3. 标题层级克制但不能取消导航：优先用H2承载大模块，必要时用少量H3、H4等子标题定位关键小节，避免无节制使用H3/H4/H5；
4. 避免滥用分割线，除非语义上确实需要强分隔，否则不要使用`---`作为排版装饰；
5. 有序列表/无序列表的每一项结尾标点使用；而非。；
