本文用于记录 Codex 里的插件、Skills 和外部连接器分别是什么，以及在真实工作流中应该怎么选择。

先给一个结论：**Codex 的基础能力是读写文件、运行命令、理解代码和完成任务；插件和 Skills 则是在这个基础上给 Codex 增加专门工具、外部数据源和可复用工作流**。

## 1. 插件、App、MCP、Skills 的区别

Codex 里的能力名称很容易混在一起，建议按下面的层次理解：

| 名称                      | 可以怎么理解             | 主要作用                                     | 典型例子                                           |
| ----------------------- | ------------------ | ---------------------------------------- | ---------------------------------------------- |
| Plugin                  | 一个可安装的能力包          | 把 Skills、MCP 工具、App 连接器、资源文件打包给 Codex 使用 | Documents、Figma、GitHub、BioRender、Readwise      |
| App / Connector         | 连接外部服务的入口          | 让 Codex 调用外部系统、账号、数据库或设计工具               | GitHub、Gmail、Canva、Figma、Readwise              |
| MCP Tool                | 可被 Codex 调用的具体工具函数 | 查询、创建、编辑、导入、渲染、搜索等具体动作                   | 搜索 GitHub PR、创建 Figma 文件、导入 Google Docs        |
| Skill                   | 面向任务的工作流说明         | 告诉 Codex 遇到某类任务时应该按什么步骤做                 | `documents`、`frontend-app-builder`、`figma-use` |
| Prompt / Default Prompt | 插件给出的示例触发方式        | 帮用户理解这个插件适合问什么                           | “Find papers in my Zotero library”             |

一个插件不一定只包含一种能力。有些插件主要提供 App 工具，有些插件主要提供 Skills，有些则同时包含工具、技能、脚本和资源。

## 2. 判断一个插件是否值得安装

选择插件时，不要只看名字，应该看它能否解决自己的高频问题。

### 2.1 优先安装的情况

- 这个插件能连接你已经在用的资料库或工作系统，例如 Zotero、Readwise、GitHub、Figma；
- 这个插件能访问普通搜索很难拿到的结构化数据，例如 PitchBook、CB Insights、GovTribe；
- 这个插件提供专业领域工作流，例如 Documents 负责 DOCX 渲染质检，Life Science Research 负责生命科学证据综合；
- 这个插件能明显减少重复劳动，例如 Canva 批量改尺寸，Figma 生成设计稿，Documents 生成正式 Word 文档；

### 2.2 可以暂缓安装的情况

- 插件依赖机构订阅或企业账号，而你当前没有对应数据权限；
- 插件只覆盖很垂直的金融、政策、医学或税务场景，暂时和当前目标无关；
- 插件只是“看起来很强”，但你没有明确的输入数据、账号、工作流或交付物；
- 插件访问的是实时金融或市场数据，但你的任务只是写普通文档或整理个人知识；

## 3. 常见生产力插件

### 3.1 Documents：正式文档生成与 DOCX 质检

Documents 是 Codex 里用于处理 Word / DOCX / Google Docs 目标文档的插件。

它能做的事包括：

- 创建新的 `.docx`、Word 文档或面向 Google Docs 的文档；
- 编辑已有 DOCX，包括局部改写、批注、红线、接受修订、删除评论；
- 做正式文档排版，例如标题层级、页边距、表格宽度、列表编号、页眉页脚；
- 把 DOCX 渲染成页面 PNG 做视觉检查，确认没有文字溢出、表格错位、分页异常；
- 为 Google Docs 准备高质量 DOCX，再通过 Google Drive 插件导入成原生 Google Docs；
- 做敏感信息脱敏、可访问性检查、评论清理等文档工程操作；

适合场景：

- 把 Markdown、会议记录或方案整理成正式 Word 报告；
- 修改合同、提案、备忘录，并保留批注或修订痕迹；
- 生成可交付给客户、评委、团队或学校的正式文档；
- 需要对 DOCX 做视觉 QA，而不只是生成文本内容；

不适合场景：

- 直接编辑普通 Obsidian Markdown 笔记；
- 只需要快速记录想法，不需要 Word 排版或交付格式；
- 需要生成网页、代码项目或可交互应用；

### 3.2 Zotero：文献库检索与引用管理

Zotero 插件用于连接本地 Zotero 文献库。

它能做的事包括：

- 在你的 Zotero 桌面文献库中查找论文；
- 导出 BibTeX；
- 给草稿插入 citation key；
- 在你允许时读取已经索引的附件文本；
- 在你确认后把新的 reference record 导入 Zotero；

适合场景：

- 写论文、文献综述、研究笔记；
- 从个人文献库里找相关论文；
- 给 LaTeX、Markdown 或正式报告补引用；
- 把知识库笔记和论文引用链路打通；

### 3.3 Readwise：个人阅读库与高亮检索

Readwise 插件用于连接 Readwise 和 Reader。

它能做的事包括：

- 语义搜索你的 highlights；
- 搜索 Reader 里保存的文章、网页和资料；
- 整理 Reader inbox；
- 组织阅读库、回顾 feed 或根据高亮生成总结；

适合场景：

- 从个人阅读历史里找上下文；
- 把书摘、文章高亮和个人知识笔记连接起来；
- 复习某个主题下过去读过的内容；
- 把“输入型知识”整理成可复用笔记；

### 3.4 LaTeX：科研与技术排版

LaTeX 插件用于编译 LaTeX，常见后端包括 Tectonic 或 TeX Live。

它能做的事包括：

- 把 `.tex` 文件编译成 PDF；
- 支持论文、数学文档、技术白皮书、学术报告等排版；
- 借助 Tectonic 自动处理部分 TeX 依赖；
- 借助 TeX Live 使用完整 TeX 生态；

适合场景：

- 写论文或技术报告；
- 需要复杂公式、交叉引用、参考文献和出版级排版；
- 与 Zotero / BibTeX 配合生成引用；

### 3.5 Scite：同行评议证据检索

Scite 插件用于生成基于同行评议研究的回答，并突出论文之间的证据关系。

它能做的事包括：

- 基于真实 scientific sources 回答问题；
- 展示某个研究被后续论文支持、反驳还是背景化引用；
- 帮助判断某个科学结论的可靠性；
- 用 citation-based ranking 优先返回更可信的研究；

适合场景：

- 做严肃科研问题判断；
- 写文献综述时区分“有人提出”和“后续研究支持”；
- 判断一个健康、心理、教育或生物医学结论是否可靠；

## 4. 外部数据与研究类插件速查

这类插件的主要价值是拓展 Codex 的信息来源和专业数据访问能力。它们通常用于连接外部数据库、市场数据、行业情报、政策资料或个人知识库，让 Codex 可以基于更明确的资料来源完成检索、分析和总结。

### 4.1 科研与知识管理

| 插件 | 主要功能 | 适合干嘛 | 注意事项 |
| --- | --- | --- | --- |
| Zotero | 查找本地 Zotero 文献库、导出 BibTeX、插入 citation key | 论文写作、文献综述、研究笔记 | 依赖你的 Zotero 库质量 |
| Life Science Research | 生命科学研究路由、实体归一化、证据综合 | 遗传学、组学、蛋白结构、药物发现、临床证据 | 适合生命科学垂直研究，不是通用搜索 |
| Scite | 基于同行评议论文回答，并展示支持、反驳、背景化引用 | 科学证据判断、文献可靠性分析 | 更适合研究判断，不适合普通资料整理 |
| Readwise | 搜索 Readwise highlights 和 Reader 内容 | 个人阅读库检索、读书笔记整理、输入沉淀 | 价值取决于个人阅读库积累 |
| LaTeX | 编译 LaTeX 文档 | 学术论文、数学文档、技术报告 | 偏排版与编译，不负责替你验证论文事实 |

### 4.2 金融市场与投研数据

| 插件 | 主要功能 | 适合干嘛 | 注意事项 |
| --- | --- | --- | --- |
| Binance | 只读访问 Binance 公共市场数据，包括价格、订单簿、成交、K 线和交易所元数据 | 加密货币行情分析、市场快照、历史价格查询 | 不访问账户，不支持交易，不构成投资建议 |
| Alpaca | 查询股票、期权和加密货币市场数据 | 股票表现比较、期权链查询、历史价格问题 | 面向市场数据和自动化分析场景 |
| CB Insights | 私营公司、融资、市场地图、竞品监控、投资备忘录 | VC、并购、创新战略、BD 找公司 | 通常依赖账号或订阅权限 |
| PitchBook | 私募资本市场结构化数据，包括公司、投资人、基金、交易、LP、人物 | 私募股权、风投、融资历史、投资组合研究 | 高价值但强依赖数据权限 |
| Daloopa | 从 SEC 文件、投资者材料和公开财报中提取基本面数据，并提供来源链接 | 财务建模、上市公司基本面分析 | 适合量化和财报分析，不是新闻工具 |
| Quartr | 上市公司 IR 数据，包括业绩会、转录稿、PPT、公告、报告和事件摘要 | 业绩会分析、KPI 追踪、管理层叙事变化 | 适合公开公司研究 |
| Morningstar | 投资和基金研究 | 基金、ETF、资产研究 | 具体可用范围取决于账号权限和插件开放能力 |
| Moody's | 信用和风险情报 | 公司信用风险、评级相关背景、风险分析 | 具体可用范围取决于账号权限和插件开放能力 |
| MT Newswires | 实时全球金融新闻和多资产市场新闻 | ticker 最新新闻、宏观事件、央行信号、评级变化 | 偏实时新闻，结论需要结合市场上下文 |
| Third Bridge | 专家网络内容和行业专家洞察 | 行业研究、竞品分析、投资研究 | 通常适合机构研究场景 |
| Cube | 查询企业实际值、预算、预测、差异等经营财务数据 | 预算差异分析、董事会汇报、财务摘要 | 依赖企业自己的 Cube 数据和权限 |
| Particl Market Research | 电商市场研究，包括商品目录、竞品、销售时间序列、产品组合 | 电商品类研究、市场 leader 识别、竞品商品分析 | 依赖 Particl 账号和数据覆盖 |

### 4.3 政策、政府、税务与信贷

| 插件 | 主要功能 | 适合干嘛 | 注意事项 |
| --- | --- | --- | --- |
| Dow Jones Factiva | 搜索 Factiva 全球新闻档案和 Dow Jones premium sources | 公司、行业、市场新闻研究 | 需要 Factiva 订阅，内容共享和使用有组织限制 |
| GovTribe | 搜索美国政府合同、award、供应商和采购机会 | 政府采购市场研究、投标准备、供应商竞争分析 | 主要面向美国公共部门采购 |
| PolicyNote | 查询全球政策、监管、法律和政府活动 | 政策监控、监管研究、政策告警工作流 | 面向政策和合规研究 |
| Taxdown | 西班牙税务问答，覆盖个人和自雇者税务问题 | 西班牙 IRPF、renta、扣除、税务义务咨询 | 地域很垂直，主要适合西班牙税务场景 |
| Tinman AI | 房贷场景承保辅助，回答资格、DTI、收入计算等问题 | 贷款官和承销员快速判断贷款方案 | 面向住房金融承保，不是通用金融顾问 |

## 5. 面向个人知识库的插件优先级

如果目标是服务个人知识库，例如沉淀阅读、论文、研究资料和长期项目上下文，不建议一上来安装大量金融和行业数据库插件。优先级可以这样排：

### 5.1 第一优先级

| 插件 | 推荐原因 |
| --- | --- |
| Zotero | 适合把论文、引用和研究笔记接入知识库，是学术知识沉淀的核心连接器 |
| Readwise | 适合把日常阅读、高亮、文章输入沉淀到知识库，是个人输入流的核心连接器 |
| Scite | 适合判断研究结论是否可靠，能提升知识笔记的证据质量 |
| Documents | 适合把知识库中沉淀的内容输出为正式 DOCX、报告、提案或 Google Docs |
| LaTeX | 适合把科研、数学、技术内容输出为论文级 PDF |

### 5.2 第二优先级

| 插件 | 推荐原因 |
| --- | --- |
| Life Science Research | 如果知识库中有生命科学、生物医药、基因组学方向内容，它会很有价值 |
| PolicyNote | 如果需要长期跟踪政策、监管、政府活动，可以接入政策研究流 |
| GovTribe | 如果未来做美国政府采购、招投标或 ToG 市场研究，可以考虑 |
| CB Insights / PitchBook | 如果需要沉淀 VC、创业、市场地图和私营公司研究，可以考虑 |

### 5.3 暂时不必优先安装

| 插件 | 原因 |
| --- | --- |
| Binance / Alpaca | 更偏实时市场数据和交易研究，和普通知识库沉淀关系不大 |
| Daloopa / Quartr / Morningstar / Moody's / MT Newswires / Third Bridge | 偏机构投研，除非有明确金融研究需求或账号权限 |
| Taxdown | 地域非常垂直，主要面向西班牙税务 |
| Tinman AI | 场景非常垂直，主要面向房贷承保 |
| Particl Market Research | 主要面向电商市场研究，除非需要做电商、品牌或零售分析 |

## 6. 实用选择策略

### 6.1 写论文或研究笔记

推荐组合：

- `Zotero`：找文献和导出 BibTeX；
- `Scite`：判断证据关系和论文可靠性；
- `LaTeX`：输出论文 PDF；
- `Documents`：输出 Word / Google Docs 版本；

典型提示词：

```text
请用 Zotero 检索文献库中关于 RAG evaluation 的论文，按主题分组，并导出可用于 LaTeX 的 BibTeX key。
```

```text
请用 Scite 核查“长期屏幕时间会显著影响青少年心理健康”这个结论，区分支持、反驳和背景化证据。
```

### 6.2 整理个人阅读和知识输入

推荐组合：

- `Readwise`：搜索 highlights 和 Reader 内容；
- `Documents`：把整理好的内容输出成正式文档；
- 本地 Markdown 笔记：作为长期知识沉淀主格式；

典型提示词：

```text
请用 Readwise 检索关于 second brain、knowledge management 和 AI memory 的高亮，整理成一篇 Markdown 笔记大纲。
```

### 6.3 做行业、市场或投资研究

推荐组合：

- `CB Insights` / `PitchBook`：私营公司和融资数据；
- `Quartr` / `Daloopa`：上市公司公开材料和基本面数据；
- `MT Newswires` / `Dow Jones Factiva`：新闻和市场动态；
- `Third Bridge`：专家观点；

典型提示词：

```text
请用 CB Insights 和 PitchBook 整理 AI coding agent 赛道的代表公司、融资轮次、投资人和竞争格局。
```

### 6.4 做政策、政府采购或合规研究

推荐组合：

- `PolicyNote`：政策与监管动态；
- `GovTribe`：美国政府合同和采购数据；
- `Dow Jones Factiva`：新闻背景；

典型提示词：

```text
请用 PolicyNote 检索最近 12 个月和 AI regulation 相关的政策更新，并按司法辖区分类。
```

## 7. 使用插件时的注意事项

### 7.1 区分数据权限和模型能力

有些插件看起来很强，但真正的关键是账号权限。没有订阅或授权时，Codex 不能凭空拿到 PitchBook、Factiva、CB Insights、Third Bridge 这类数据。

### 7.2 区分只读数据和可执行操作

例如 Binance 插件的定位是公共、只读市场数据，不访问用户账户，也不支持交易或转账。使用金融插件时，应该把它们当成研究和分析工具，而不是交易执行工具。

### 7.3 区分插件能力说明和个人选择建议

插件官方描述可以作为能力边界参考；本文中的优先级和安装建议只是面向个人知识库工作流的取舍，不代表插件官方承诺。

### 7.4 对实时数据保持日期意识

市场数据、新闻、政策、评级和金融研究都可能快速变化。写入知识库时建议标注检索日期，例如“截至 2026-05-25”。

### 7.5 不要把所有问题都交给插件

插件适合补足外部数据和专业工作流，但 Codex 的核心价值仍然是把上下文、文件、命令、代码和交付物串起来。插件应该服务于任务，而不是反过来决定任务。

## 8. 一句话记忆

Codex 插件是“外部能力包”，Skills 是“任务工作流说明”。

对个人知识库来说，优先安装能连接知识输入和高质量输出的插件，例如 Zotero、Readwise、Scite、Documents 和 LaTeX；金融、政策、行业数据库类插件则按具体研究需求再安装。
