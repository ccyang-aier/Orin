# 00. 架构初探及教程大纲规划

## 1. 系列定位

AIBrix 是面向大模型推理服务的云原生 MaaS 平台能力集合。它不只解决单个模型服务如何启动的问题，更关注模型服务在 Kubernetes 环境中的生命周期管理、弹性治理、缓存感知、请求路由、分布式部署和生产化运行。

本系列的目标不是简单罗列 AIBrix 的源码模块，而是围绕“一个模型如何被部署、调度、扩缩容、缓存命中、路由转发并稳定运行”这条主线，逐步拆解 AIBrix 的架构设计与底层运作原理。

## 2. 阅读背景

读者最好已经具备以下基础：

- 理解 Kubernetes 中 Pod、Deployment、Service、CRD、Controller、Reconcile 等基本概念；
- 理解 LLM Serving 的基本挑战，例如模型加载、GPU 资源、并发请求、吞吐、延迟和多副本服务；
- 对 vLLM、Ray、Gateway API、Envoy、KV Cache 等相关技术有基本印象即可，不要求预先掌握内部实现。

## 4. 写作原则

后续章节应优先围绕架构主线组织，而不是按源码目录机械展开。每一章都应回答三个问题：

- 这个能力要解决什么 MaaS 或 LLM Serving 问题；
- AIBrix 用哪些抽象、控制器或运行时组件解决它；
- 这个能力如何与其他章节中的生命周期、CRD、工作负载、弹性、缓存和路由机制连接起来。

如果某一章内容过大，可以按机制边界拆成上下篇，但仍应保持同一条学习主线，避免把系列写成零散模块笔记。
