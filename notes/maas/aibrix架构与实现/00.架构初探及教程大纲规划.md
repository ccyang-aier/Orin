# 00. 架构初探及教程大纲规划

## 1. 系列定位

AIBrix 是面向大模型推理服务的云原生 MaaS 平台能力集合。它不只解决单个模型服务如何启动的问题，更关注模型服务在 Kubernetes 环境中的生命周期管理、弹性治理、缓存感知、请求路由、分布式部署和生产化运行。

本系列的目标不是简单罗列 AIBrix 的源码模块，而是围绕“一个模型如何被部署、调度、扩缩容、缓存命中、路由转发并稳定运行”这条主线，逐步拆解 AIBrix 的架构设计与底层运作原理。

## 2. 阅读背景

读者最好已经具备以下基础：

- 理解 Kubernetes 中 Pod、Deployment、Service、CRD、Controller、Reconcile 等基本概念；
- 理解 LLM Serving 的基本挑战，例如模型加载、GPU 资源、并发请求、吞吐、延迟和多副本服务；
- 对 vLLM、Ray、Gateway API、Envoy、KV Cache 等相关技术有基本印象即可，不要求预先掌握内部实现。

## 3. 大纲结构

### 3.1 AIBrix 要解决什么问题

从 LLM Serving 的基础问题切入，说明为什么只依赖普通 Deployment 和 Service 很难支撑 MaaS 平台能力，并进一步解释 AIBrix 在模型服务生命周期、控制面抽象、请求治理和生产运维中的定位。

### 3.2 CRD 中心化架构

建立 AIBrix 的第一层全景认知：为什么它围绕 CRD 组织核心能力，各类 CRD 分别表达什么领域语义，以及这些 CRD 如何与 Kubernetes 原生资源、控制器和运行时工作负载协同。

### 3.3 模型生命周期与基础推理工作负载

围绕模型从声明、部署、运行到更新的生命周期，拆解基础推理工作负载的机制和作用。重点关注单模型、单卡或常规多副本服务的部署视图、控制流和运行时结构。

### 3.4 复杂推理部署形态

继续展开更复杂的推理服务形态，例如分布式推理、PD 分离、多实例、多角色工作负载等。该部分重点解释不同形态为什么需要不同的编排抽象，以及它们如何影响控制面、数据面和故障恢复。

### 3.5 弹性机制

围绕扩缩容、自愈、容量治理和工作负载恢复展开，解释 AIBrix 如何在 Kubernetes 基础弹性能力之上增加面向模型服务的领域语义。

### 3.6 KVCache 系统

讲解 AIBrix 的 KVCache 抽象、缓存拓扑维护、缓存感知能力，以及它们如何服务于请求调度、亲和路由和推理性能优化。

### 3.7 路由系统

结合 Gateway、Envoy、Router 和多种路由策略，拆解请求如何进入 AIBrix，如何选择后端模型实例，以及缓存、负载、可用性等信号如何影响路由决策。

### 3.8 扩展能力与特殊架构

整理 AIBrix 中不属于主链路但影响平台能力边界的设计，例如异构资源、特性开关、扩展控制器、特殊部署模式和周边集成。

### 3.9 生产化设计

从生产环境视角回看 AIBrix 的可观测性、稳定性、运维边界、故障排查和平台治理能力，说明它如何从“能部署模型”走向“能长期运营模型服务”。

### 3.10 全局串联

将前面章节重新串联为完整知识体系：从一次模型声明开始，跟踪控制器、工作负载、缓存、路由、弹性和可观测性的协作关系，形成对 AIBrix 架构与底层运作原理的整体理解。

## 4. 写作原则

后续章节应优先围绕架构主线组织，而不是按源码目录机械展开。每一章都应回答三个问题：

- 这个能力要解决什么 MaaS 或 LLM Serving 问题；
- AIBrix 用哪些抽象、控制器或运行时组件解决它；
- 这个能力如何与其他章节中的生命周期、CRD、工作负载、弹性、缓存和路由机制连接起来。

如果某一章内容过大，可以按机制边界拆成上下篇，但仍应保持同一条学习主线，避免把系列写成零散模块笔记。
