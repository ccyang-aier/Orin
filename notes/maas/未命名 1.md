## 第一页｜易用性全景与当前进展

### 易用性定义

> **易用 ≠ 体验好**
> 易用解决的是：**用户在每个阶段能否低门槛地用起来、用得顺**
> 体验好解决的是：响应是否够快、服务是否稳定、结果是否满意

两者互为补充，共同构成平台竞争力的核心支柱。

---

### 易用性核心维度

围绕 FusionOne AI 核心特性模块，易用性划分为以下五大方向：

| #   | 方向            | 核心目标                           |
| --- | ------------- | ------------------------------ |
| 1   | **观测与排障易用**   | 可观测性完备闭环，故障定位自动化、智能化，加速特性价值白盒化 |
| 2   | **部署易用**      | 推理服务配置智能化，降低用户心智负担与理解成本        |
| 3   | **模型全生命周期易用** | 覆盖模型管理、选型、评估、体验全链路，降低决策与操作成本   |
| 4   | **应用构建易用**    | AI 应用（知识库、智能体）低门槛构建，面向非开发者友好   |
| 5   | **跨模块流转易用**   | 消除模块间断层，用户完整旅程顺滑贯通             |

---

### 当前进展总览

**重点方向：观测与排障易用（可观测性建设）**

| 特性 | 状态 | 说明 |
|------|------|------|
| 推理服务端到端拓扑 | ✅ 已上线 | 模型→实例→卡三层贯通，应用层适配进行中 |
| 物理面板可视 | ✅ 已上线 | 硬件资源层可视 |
| 硬件资源监控 | ✅ 已上线 | GPU 利用率、显存等核心指标 |
| 推理服务全链路指标监控 | ✅ 已上线 | TTFT、TPOT、请求数、Tokens 数 |
| 推理加速特性价值指标监控 | ✅ 已上线 | KVCache 卸载利用率、推测解码加速比等价值指标 |

---

**其它方向现状**

- **部署易用**：支持自定义推理引擎与 Wings 解耦推理引擎向导式部署；但部署流程仍较繁琐，参数配置复杂，用户心智负担高，待持续优化
- **模型全生命周期易用**：模型管理、模型广场、模型体验/竞技场、模型评估各模块已具备基础能力，但模块间串联不足，用户需在多入口间频繁跳转
- **应用构建易用**：知识库、智能体等 AI 应用能力已上线，面向业务用户的低门槛引导与效果闭环验证机制待建设
- **跨模块流转易用**：各模块独立演进为主，统一资源视图与跨模块用户旅程尚未系统性规划

## 第二页｜观测与排障易用 — 详细说明

### 目标

> 可观测性完备且闭环，故障定位自动化、智能化，加速特性价值白盒化

---

### 当前进展

#### 1. 可视化能力

**推理服务端到端拓扑**

```
[应用层]  知识库 / 智能体         ← 适配中，暂未覆盖
    ↓
[服务层]  推理服务                ✅ 已覆盖
    ↓
[实例层]  推理实例                ✅ 已覆盖
    ↓
[硬件层]  GPU 卡                  ✅ 已覆盖
```

- 实现从模型到实例到卡的三层可视化贯通
- 故障链路不再是黑盒，定位效率从"不知道哪层出问题"提升至**分钟级精准到卡级**
- 物理面板可视，硬件资源状态一目了然

#### 2. 性能监控

**监控对象**：推理服务 + 推理实例双层监控

**指标体系**：

| 类别 | 指标 | 价值 |
|------|------|------|
| 时延指标 | TTFT（首 Token 时延）、TPOT（每 Token 生成时延） | 用户侧体感质量量化 |
| 吞吐指标 | 请求数、Tokens 数 | 服务负载全貌 |
| 加速特性价值指标 | KVCache 卸载利用率、推测解码加速比 | 加速收益从黑盒变白盒 |
| 硬件资源指标 | GPU 利用率、显存占用 | 资源健康度感知 |

---

### 后续演进

**短期（近 1-2 个迭代）**

- 应用层（知识库、智能体）纳入 E2E 拓扑，实现从应用到卡的**全链路四层贯通**
- 监控告警支持用户**自助配置阈值与通知渠道**，不依赖平台介入
- 异常告警增加**根因自动标注**，告警信息从"展示数据"升级为"给出可操作建议"

**中期**

- 故障根因**智能诊断**：系统自动关联拓扑异常与监控指标，输出结构化排障报告
- 加速特性价值**可量化对比**：开启/关闭加速前后效果对比可视化，用户可直观感知收益

---

### 竞争力构筑方向

```
当前差异化                        持续构筑
──────────────────────────────────────────────────────
✅ 自研引擎指标直接透出            → 加速特性价值智能分析与推荐
✅ 三层拓扑精准到卡级              → 四层贯通（含应用层）
✅ 推理全链路指标完整              → 根因自动诊断，排障无需人工介入
```

> **核心竞争力**：依托自研推理引擎的深度指标采集能力，实现竞品无法复制的加速特性白盒化可观测体系。

## 第三页｜部署易用 — 详细说明

### 目标

> 推理服务配置智能化，降低用户心智负担与理解成本，让用户聚焦业务而非基础设施

---

### 当前进展

**已有能力**

- 支持**自定义推理引擎**部署
- 支持 **Wings 解耦推理引擎**向导式部署，用户无需关心底层引擎细节

**现存问题**

| 问题              | 影响                     |
| --------------- | ---------------------- |
| 部署流程步骤多、参数项复杂   | 新用户上手成本高，依赖文档或客服引导     |
| 实例规格、并发数等参数缺乏引导 | 用户易配置不当，导致资源浪费或性能不达预期  |
| 部署前无费用/资源预估     | 成本不透明，用户决策困难           |
| 部署状态反馈不够清晰      | 部署中/失败时用户不知道"卡在哪、怎么解决" |

---

### 后续演进

**短期**

- 部署参数**智能推荐**：根据模型规格、业务场景自动推荐实例规格与并发配置，降低选择成本
- 部署前**资源与费用预估**：让用户在提交前了解大致消耗，消除成本黑盒
- 部署失败信息**结构化提示**：明确告知失败原因与解决建议，而非只展示错误码

**中期**

- 向导式部署升级为**场景化模板**：按"高并发推理 / 低延迟对话 / 批量离线处理"等场景提供开箱即用配置
- 支持**灰度切流**：新服务上线时支持按比例分流，降低全量上线风险
- 部署后**配置优化建议**：基于实际运行指标反向提供配置调优建议

---

### 竞争力构筑方向

```
当前基础                          持续构筑
──────────────────────────────────────────────────────
✅ 向导式部署降低引擎理解成本      → 场景化模板，零配置快速上线
✅ Wings 解耦引擎支持              → 智能推荐 + 费用预估，决策透明
                                   → 配置-效果闭环，部署即可调优
```

> **核心竞争力**：结合自研引擎能力，平台可做到"懂模型、懂资源"的智能部署引导，而非通用 IaaS 式的参数填写。

## 第四页｜模型全生命周期易用 — 详细说明

> 说明：此方向覆盖范围不止"选型"，而是贯穿模型管理、选型决策、效果评估、体验调试的完整生命周期。

### 目标

> 用户在模型全生命周期各阶段均能低门槛操作，模型管理→选型→评估→体验各环节串联贯通，而非孤立模块

---

### 当前进展

| 模块 | 现状 |
|------|------|
| 模型管理 | 基础管理能力完备，模型版本、状态管理可用 |
| 模型广场 | 模型列表与基础信息展示，能力描述以参数规格为主 |
| 模型体验 / 竞技场 | 支持在线体验与多模型横向对比 |
| 模型评估 | 支持主流 Benchmark 评估，私有数据集评估能力待完善 |

**现存问题**：四个模块独立存在，用户需在多入口间反复跳转，体验结果无法直接流转至部署。

---

### 后续演进

**短期**

- 模型广场增加**场景化推荐入口**（如：适合 RAG / 适合长文本 / 适合代码生成）
- 模型能力描述从"参数规格"升级为**结构化能力卡片**（优势场景、局限说明、推荐配置）

**中期**

- **体验→评估→部署一键流转**：在体验/竞技场中满意的模型，直接发起评估任务或创建推理服务，消除模块断层
- 支持用户上传**私有业务数据集**做评估，而非只能跑公开 Benchmark
- 模型切换提供**效果回归工具**：换模型前后指标对比，帮助用户有信心做决策

---

### 竞争力构筑方向

```
当前基础                          持续构筑
──────────────────────────────────────────────────────
✅ 竞技场多模型横向对比            → 体验-评估-部署全链路串联
✅ 主流 Benchmark 评估能力         → 私有数据集评估，贴近真实业务
                                   → 场景化选型引导，降低决策成本
```
