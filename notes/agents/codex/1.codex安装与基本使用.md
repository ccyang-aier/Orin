# Codex 安装与基本使用

## Windows 本地安装与验证

本机已验证 Codex CLI 可用：

```powershell
codex --version
```

当前输出：

```text
codex-cli 0.130.0-alpha.5
```

这说明当前 Windows 环境里已经可以直接从 PowerShell 调用 `codex` 命令。结合本机环境排查，`node` 来自 Codex 桌面版自带运行时，而 `npm`、`winget` 当前不在 PATH 中；因此这台机器当前更像是通过 Codex 桌面版或其运行时一并提供了 CLI，而不是通过系统级 npm 全局安装。

## 官方安装路径

OpenAI 官方仓库的推荐安装方式是使用 npm 全局安装：

```powershell
npm install -g @openai/codex
```

安装后验证：

```powershell
codex --version
```

如果已安装 Codex CLI，也可以用内置升级命令：

```powershell
codex --upgrade
```

官方仓库同时提供 GitHub Release 二进制包；在没有 npm，或 npm 的全局路径没有配置好的机器上，可以从 release 页面下载适合平台的压缩包，解压后把可执行文件所在目录加入 PATH。

参考来源：

- [openai/codex GitHub 仓库](https://github.com/openai/codex)；
- [OpenAI Help Center: Codex CLI Getting Started](https://help.openai.com/en/articles/11096431-openai-codex-cli-getting-started)；

## Windows 注意点

官方文档曾提示 Windows 支持可能需要 WSL；不过本机当前在原生 Windows PowerShell 中已经可以执行 `codex`。如果另一台 Windows 机器遇到 `codex`、`npm` 或 `winget` 命令不可用，优先按下面顺序排查：

1. 执行 `codex --version`，确认是否已经随 Codex 桌面版或其他安装方式可用；
2. 执行 `Get-Command codex`，查看 `codex` 命令实际来自哪个路径；
3. 执行 `node --version` 和 `npm --version`，确认 Node.js 与 npm 是否都在 PATH；
4. 如果有 Node.js 但没有 npm，建议重新安装官方 Node.js LTS，或使用 GitHub Release 二进制包；
5. 如果原生 Windows 行为异常，再考虑在 WSL2 中安装并使用 Codex CLI；

## 基本启动方式

在目标项目根目录启动：

```powershell
cd C:\AIWorks\Orin
codex
```

首次启动时按提示登录。官方推荐使用 ChatGPT 账号登录，以便使用 Plus、Pro、Business、Edu 或 Enterprise 计划中的 Codex 能力；也可以配置 OpenAI API key 使用。

常用模式：

```powershell
codex --suggest
codex --auto-edit
codex --full-auto
```

- `--suggest`：默认模式，适合阅读代码、提出建议、需要人工确认修改的场景；
- `--auto-edit`：允许自动编辑文件，但执行 shell 命令前仍会确认；
- `--full-auto`：允许在受限沙箱里更自主地读写与执行命令，适合长任务或原型验证；

## 在 Orin 中使用

建议在 Orin 根目录启动 Codex，让它能读取 `AGENTS.md` 中的项目说明与写作约束：

```powershell
cd C:\AIWorks\Orin
codex
```

适合交给 Codex 的任务包括：

- 整理 `inbox/` 中的临时材料；
- 为 `notes/` 补充结构化 frontmatter；
- 编写或维护 `code/` 下的自动化脚本；
- 设计 `rag/`、`database/` 相关的数据结构；
- 在任务完成后执行 git 检查与英文 commit；

使用习惯：

1. 先明确目标文件或目录，减少误改范围；
2. 让 Codex 在修改前说明准备编辑的内容；
3. 修改后要求它运行必要的验证命令；
4. 确认无误后提交 git commit，commit message 使用英文；
