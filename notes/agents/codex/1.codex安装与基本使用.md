# 1. Codex 安装与基本使用

## 1.1 Windows 本地安装与验证

Codex CLI 推荐通过 npm 全局安装。安装前先确认 Node.js 与 npm 可用：

```powershell
node --version
npm --version
npm config get prefix
```

安装命令：

```powershell
npm install -g @openai/codex@latest
```

安装后验证：

```powershell
codex --version
Get-Command codex
```

如果安装成功，`codex --version` 会输出类似：

```text
codex-cli 0.130.0
```

在 Windows 中，npm 全局安装通常会生成以下命令入口，实际路径以 `npm config get prefix` 输出为准：

```text
<npm-prefix>\codex
<npm-prefix>\codex.cmd
<npm-prefix>\codex.ps1
```

## 1.2 官方安装路径

OpenAI 官方仓库推荐使用 npm 全局安装：

```powershell
npm install -g @openai/codex
```

如果已安装 Codex CLI，也可以使用升级命令：

```powershell
codex --upgrade
```

官方仓库也提供 GitHub Release 二进制包；在没有 npm，或 npm 全局路径没有配置好的机器上，可以从 release 页面下载适合平台的压缩包，解压后把可执行文件所在目录加入 PATH；

参考来源：

- [openai/codex GitHub 仓库](https://github.com/openai/codex)；
- [OpenAI Help Center: Codex CLI Getting Started](https://help.openai.com/en/articles/11096431-openai-codex-cli-getting-started)；

## 1.3 Windows 排查要点

如果在 PowerShell 中执行 `codex --version` 提示无法识别命令，按下面顺序排查：

1. 确认 npm 是否可用：`npm --version`；
2. 查看 npm 全局目录：`npm config get prefix`；
3. 确认 npm 全局目录是否在 PATH 中：`[Environment]::GetEnvironmentVariable('Path','User')`；
4. 重新安装 Codex CLI：`npm install -g @openai/codex@latest`；
5. 查看命令解析路径：`Get-Command codex`；
6. 重新打开 PowerShell，再执行 `codex --version`；
7. 如果原生 Windows 行为异常，再考虑在 WSL2 中安装并使用 Codex CLI；

注意：Codex app、IDE 插件或自动化代理可能有自己的运行时、沙盒和 PATH。安装与 PATH 类问题必须以用户手动打开的真实终端为准；

## 1.4 基本启动方式

在目标项目根目录启动：

```powershell
cd C:\AIWorks\Orin
codex
```

首次启动时按提示登录。官方推荐使用 ChatGPT 账号登录，以便使用 Plus、Pro、Business、Edu 或 Enterprise 计划中的 Codex 能力；也可以配置 OpenAI API key 使用；

常用模式：

```powershell
codex --suggest
codex --auto-edit
codex --full-auto
```

- `--suggest`：默认模式，适合阅读代码、提出建议、需要人工确认修改的场景；
- `--auto-edit`：允许自动编辑文件，但执行 shell 命令前仍会确认；
- `--full-auto`：允许在受限沙箱里更自主地读写与执行命令，适合长任务或原型验证；

## 1.5 在 Orin 中使用

建议在 Orin 根目录启动 Codex，让它能读取 `AGENTS.md` 中的项目说明与写作约束：

```powershell
cd C:\AIWorks\Orin
codex
```

适合交给 Codex 的任务包括：

- 整理 `inbox/` 中的临时材料；
- 为 `notes/` 补充结构化 frontmatter；
- 编写或维护 `code/` 下的自动化脚本；
- 设计 `rag/`、`database/` 相关的数据结构；
- 在任务完成后执行 git 检查与英文 commit；

使用习惯：

1. 先明确目标文件或目录，减少误改范围；
2. 让 Codex 在修改前说明准备编辑的内容；
3. 修改后要求它运行必要的验证命令；
4. 确认无误后提交 git commit，commit message 使用英文；
