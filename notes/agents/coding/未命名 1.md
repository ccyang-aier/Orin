# 第二章：环境搭建 —— 从零安装与认证

> **本章目标**：完成 Codex 所有形态的安装与认证，并成功发送第一条消息。预计用时 **10～15 分钟**。本章会覆盖 macOS、Linux、Windows（WSL）三种系统环境下的安装步骤，以及 ChatGPT 账号和 API Key 两种认证方式。

---

## 2.1 前置准备：确认账号与系统要求

### 账号要求

在安装任何东西之前，先确认你有可用的账号：

- **推荐**：ChatGPT **Plus / Pro / Business / Edu / Enterprise** 任意一种套餐
- **备选**：OpenAI **API Key**（仅支持 CLI 和 IDE Extension，不支持 Cloud Agent）

如果你还没有订阅，前往 [chat.openai.com](https://chat.openai.com) 升级账号。免费版无法使用 Codex。

### 系统要求

| 形态 | macOS | Linux | Windows |
|---|---|---|---|
| CLI | ✅ 原生支持 | ✅ 原生支持 | ⚠️ 建议 WSL，实验性原生支持 |
| IDE Extension | ✅ | ✅ | ✅（推荐配合 WSL 使用）|
| Cloud Agent | ✅（浏览器访问）| ✅（浏览器访问）| ✅（浏览器访问）|
| App | ✅ | ❌ | ✅ |

> **Windows 用户注意**：强烈建议先安装 WSL（Windows Subsystem for Linux），再在 WSL 环境中安装 CLI。这样可以获得完整的沙箱和审批功能。Windows 原生 CLI 支持还处于实验阶段。

## 2.2 安装 Codex CLI

CLI 是 Codex 最核心的入口，建议所有用户都安装。

### macOS / Linux 安装

**方式一：npm 安装（推荐）**

确保你已安装 Node.js 18+，然后运行：

```bash
npm install -g @openai/codex
```

**方式二：Homebrew 安装（macOS 专属）**

```bash
brew install codex
```

安装完成后，验证版本：

```bash
codex --version
```

看到版本号输出（如 `0.1.x`）即表示安装成功。

### Windows 安装（WSL 方式，推荐）

**第一步：安装 WSL**

以管理员身份打开 PowerShell，运行：

```powershell
wsl --install
```

默认会安装 Ubuntu。安装完成后重启电脑。

**第二步：进入 WSL 并安装 Node.js**

```bash
# 进入 WSL 终端后
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs
```

**第三步：安装 Codex CLI**

```bash
npm install -g @openai/codex
```

**第四步：验证**

```bash
codex --version
```

### Windows 原生安装（实验性，不推荐）

如果你确实不想用 WSL，可以在 Windows 原生 PowerShell 中：

```powershell
npm install -g @openai/codex
```

但需要注意：Windows 原生模式下沙箱机制不完整，且部分功能不可用。如果使用 Windows 原生模式，需要在配置文件中设置：

```toml
# ~/.codex/config.toml（Windows 路径：%USERPROFILE%\.codex\config.toml）
[windows]
sandbox = "elevated"   # 推荐，需要管理员权限
# sandbox = "unelevated"   # 无管理员权限时的备用选项
```

## 2.3 CLI 认证：首次登录

安装完成后，在终端运行：

```bash
codex
```

首次启动会进入认证流程，你有两种选择：

### 方式 A：ChatGPT 账号登录（推荐）

1. CLI 会在终端显示一个认证提示
2. 按提示在浏览器中打开 OAuth 授权页面
3. 用你的 ChatGPT 账号登录并授权
4. 回到终端，认证完成，自动进入 TUI 界面

```
$ codex
✓ 正在打开浏览器进行认证...
✓ 认证成功，欢迎使用 Codex！
```

**这种方式的好处**：使用你套餐内包含的额度，无需额外付费，配置最简单。

### 方式 B：API Key 认证

如果你想用 API Key（如嵌入 CI 系统），有两种配置方式：

**方式 B1：环境变量（推荐用于 CI）**

```bash
export OPENAI_API_KEY="sk-..."
codex
```

**方式 B2：写入配置文件（本地持久化）**

```toml
# ~/.codex/config.toml
api_key = "sk-..."
```

> **安全提醒**：不要把 API Key 提交到代码仓库。CI 场景下建议使用 Secrets 管理（如 GitHub Secrets）注入环境变量，而不是写入配置文件。

### 验证认证是否成功

认证完成后，你会看到 Codex 的 TUI 全屏界面，顶部显示当前模型名称。输入一句话测试：

```
你好，Codex！请告诉我你当前使用的模型是什么？
```

如果收到正常回复，认证成功。

## 2.4 安装 IDE Extension

### VS Code

**方式一：通过扩展市场**

1. 打开 VS Code
2. 按 `Ctrl+Shift+X`（macOS: `Cmd+Shift+X`）打开扩展面板
3. 搜索 **"Codex"**（发布者为 OpenAI）
4. 点击安装

**方式二：直接下载**

前往 VS Code Marketplace 的 Codex 页面下载安装。

### Cursor

1. 打开 Cursor
2. 进入 Extensions 面板
3. 搜索并安装 **"Codex"**

### Windsurf

与 VS Code 步骤相同，在 Windsurf 的扩展面板中搜索安装。

### VS Code Insiders

操作与 VS Code 相同，在 Insiders 版本中安装。

### IDE Extension 登录

安装后，在编辑器侧边栏找到 Codex 图标（可能在折叠的扩展区域中，可拖拽到右侧边栏方便使用）。

点击图标，会提示登录：

- 选择 **"Sign in with ChatGPT"**，完成 OAuth 流程（与 CLI 共享认证状态）
- 或者输入 API Key

### Windows + VS Code 特别配置

在 Windows 上使用 VS Code + Codex 时，建议开启 WSL 模式以获得完整沙箱能力：

```json
// VS Code settings.json
{
  "chatgpt.runCodexInWindowsSubsystemForLinux": true
}
```

这样 IDE Extension 会在 WSL 内运行命令，继承 Linux 沙箱语义。

## 2.5 访问 Codex Cloud

Codex Cloud 无需安装，直接用浏览器访问即可。

### 第一步：登录

前往 **[chatgpt.com/codex](https://chatgpt.com/codex)**，用你的 ChatGPT 账号登录。

### 第二步：连接 GitHub 仓库

Codex Cloud 的核心是与 GitHub 集成，首次使用需要配置**环境（Environment）**：

1. 进入环境设置页面（界面上有引导）
2. 点击 **"Connect GitHub Repository"**
3. 授权 OpenAI GitHub App 访问你的仓库
4. 选择你想要连接的仓库和分支
5. 配置环境依赖（可选）：
   - 指定安装命令（如 `npm install`、`pip install -r requirements.txt`）
   - 指定可访问的外部域名（如你的包管理器镜像）

### 第三步：发起第一个任务

环境配置完成后，在输入框中描述你想让 Codex 做什么，然后点击发送。

Codex 会在云端容器中：
1. 克隆你的仓库
2. 安装依赖
3. 执行任务
4. 生成代码变更

你可以实时查看日志，任务完成后审查 diff，然后一键创建 PR。

> **注意**：Codex Cloud 需要仓库连接，如果你只想快速体验，建议先用 CLI 模式。

## 2.9 常见安装问题排查

### 问题一：`codex: command not found`

**原因**：npm 全局 bin 目录不在 PATH 中。

**解决**：
```bash
# 查看 npm 全局 bin 路径
npm bin -g

# 将其添加到 PATH（以 ~/.zshrc 为例）
echo 'export PATH="$(npm bin -g):$PATH"' >> ~/.zshrc
source ~/.zshrc
```

### 问题二：macOS 上提示"无法验证开发者"

**解决**：
```bash
# 在系统偏好设置 → 安全性与隐私 中允许运行
# 或者
xattr -d com.apple.quarantine /path/to/codex
```

### 问题三：认证后 TUI 闪退

**原因**：终端模拟器兼容性问题。

**解决**：
- macOS：推荐使用 iTerm2 或 macOS 自带终端
- Linux：推荐使用 gnome-terminal、alacritty、kitty
- Windows WSL：推荐使用 Windows Terminal

### 问题四：API Key 认证提示 "Unauthorized"

**检查**：
1. 确认 API Key 格式正确（以 `sk-` 开头）
2. 确认账号有 API 访问权限（需要在 [platform.openai.com](https://platform.openai.com) 开通）
3. 确认账号余额充足

### 问题五：Windows 原生安装报沙箱错误

**解决**：切换到 WSL 模式，或在 `config.toml` 中设置：

```toml
[windows]
sandbox = "unelevated"
```
